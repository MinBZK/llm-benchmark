## 🤝 Code of Conduct

Dit project en iedereen die eraan deelneemt, valt onder onze Code of Conduct. Door deel te nemen, wordt van je verwacht
dat je je aan deze code houdt.

## ❓ Ik heb een vraag

Als je een vraag hebt over het project, kun je het volgende doen:

1. Controleer of je vraag al is beantwoord in bestaande [Issues](https://github.com/yourusername/llm-benchmark/issues)
2. Maak een nieuw Issue aan als je vraag nog niet is gesteld

## 🔍 Ik wil een benchmark bijdragen

Fantastisch! Volg deze stappen om een nieuwe benchmark toe te voegen:

1. Fork het repository 🍴
2. Maak een nieuwe directory aan in de `benchmarks` map 📁
3. Voeg je benchmark code, data en documentatie toe 💻
4. Schrijf een duidelijke README.md die de volgende elementen bevat:
    - Doel van de benchmark 🎯
    - Methodologie 🔬
    - Implementatiedetails ⚙️
    - Instructies voor gebruik 📋
5. Maak een Pull Request aan 🚀

Zorg ervoor dat je benchmark:

- ✅ Goed gedocumenteerd is
- ✅ Reproduceerbaar is
- ✅ Duidelijke instructies bevat
- ✅ Voldoet aan de codeerstijl van het project

## 🐛 Ik wil een fout melden

Als je een fout hebt gevonden, maak dan een Issue aan met:

- Een duidelijke beschrijving van het probleem
- Stappen om het probleem te reproduceren
- Verwacht gedrag versus werkelijk gedrag
- Screenshot (indien van toepassing)

## 💡 Ik wil een verbetering voorstellen

Suggesties voor verbeteringen zijn welkom! Je kunt:

1. Een Issue aanmaken met je suggestie
2. Een Pull Request maken met de implementatie van je voorstel

### 🔄 Een Pull Request maken

1. Fork het repository 🍴
2. Maak je wijzigingen ✏️
3. Test je wijzigingen 🧪
4. Commit je wijzigingen met een duidelijke commit message 📝
5. Push naar je fork 🚀
6. Maak een Pull Request aan 🔄
7. Wacht op beoordeling en eventuele feedback 👀

Bedankt voor je bijdrage aan het LLM Benchmark project! 🎉
